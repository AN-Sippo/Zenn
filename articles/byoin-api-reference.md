# Byoin Reference

## Overview

byoinは日本語で病院と呼ばれ、以下のような機能を提供している。

- 体調不良の際に訪れると”医師”という職業の人が診察をして、治療や薬を処方してくれる
- 以前に診断された病気に対する定期的な診察

“医師”という職業は、要するに”医学”という学問のwrapperであり、特別な勉強をしなくても、自然言語を介して医学の恩恵を享受できるように設計されている。

Byoinで提供しているもっとも代表的なAPIを１つ、以下に示す。

## “診療”API

Byoinで特に何も指定しなかったとき、デフォルトの挙動が”診療”APIである。

```ts
type 診療Params = 今回の訴え | 過去の診療結果;

type 診療 = (params: Array<診療Params>) => { 診断?; 治療?; 説明 };
```

診療Paramsには過去の診療結果も許可され、2回目以降の呼び出しでは、過去のログと今回の訴えから、「過去の診療結果」を自動的に引数に含める。(なお、ここで使用している過去のログを残す仕組みを”カルテ”と呼ぶ。詳しくはリファレンスを参照してほしい)

この”カルテ’という仕組みのおかげで、1度目の呼び出しで納得の行く値が返ってこなかった場合でも、複数回呼び出すことで、返り値の精度が良くなることが期待される。

複数回の呼び出しで精度が良くなることこそ、”診療”APIの驚くべき特徴の１つであり、ぜひ活用してほしい。

なお、複数回呼び出す時の注意点は、”前回の診療を受けて体調がどう変化したか・まだ伝えられていない情報”を今回の訴えに含めることだ。

```ts
// 1回目の呼び出し
診療(["お腹の調子が悪いです"]);
// -> 整腸剤で様子を見ましょう

// N回目(半年後)の呼び出し
診療(["良くなりません", "排便すると少し楽になります"]);
// -> 過敏性腸症候群かも。詳しく検査してから、生活も含めて経過を見ていきましょう
```

注目すべきは、N回目の呼び出しだ。返り値が変化していることに気がついてほしい。

N回目の呼び出しは、内部的に以下の呼び出しに書き換えられている。

```ts
_診療([
  "排便で軽快する",
  "半年間継続する",
  "整腸剤単独では効果が薄い",
  "お腹の調子が悪い",
]);
```

この仕組みによって、複数回の呼び出しで精度が良くなる機能が実現されている。

注意してほしいのは、異なるByoinインスタンスでは、過去のログつまり、”カルテ”が常には共有されていない（我々はこの問題に取り組んでいるが、まだ完全には解決していない。詳しくは医療DXなどを参照）ことである。

1回目の呼び出しで期待する値が返ってこなかったとき、別のByoinインスタンスを探すことよりむしろ、同じByoinを複数回呼び出すことを検討してほしい。

何度も呼び出したにも関わらず、納得の行く値が返らないときに初めて別のByoinインスタンスを探すことを検討すべきだ。

“診療”APIは、その方が早く期待通りの値を得られるよう設計されており、この設計思想を”かかりつけ医”と呼ぶ。

:::message
注意点

多くの場合、同じByoinインスタンスに対する複数回の”診療”呼び出しが適しているが、例外もある。

症状が突然悪化したり、危険なサイン——今回の例で言うと便が赤い・黒いなど——の症状が出ていたら、こだわらず手近なインスタンスにアクセスすべきだ。

しかしこのような緊急時にも、カルテが1箇所に貯まっている恩恵は大きい。
初めて呼ぶインスタンスでも初回精度が上がる。それを支えるのが"紹介"というシステムだ。

:::
